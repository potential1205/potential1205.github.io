import java.util.*;

public class BinarySearchUtils {

    // 1) 기본 이분 탐색: 값이 있으면 인덱스, 없으면 -1
    public static int binarySearch(int[] a, int target) {
        int l = 0, r = a.length - 1;
        while (l <= r) {
            int m = l + (r - l) / 2;
            if (a[m] == target) return m;
            if (a[m] < target) l = m + 1;
            else r = m - 1;
        }
        return -1;
    }

    // 2) lower_bound: target 이상이 처음 나타나는 위치(삽입 위치)
    //   예) [1,2,4,4,5], target=4 -> 2
    public static int lowerBound(int[] a, int target) {
        int l = 0, r = a.length; // [l, r)
        while (l < r) {
            int m = l + (r - l) / 2;
            if (a[m] < target) l = m + 1;
            else r = m;
        }
        return l; // 삽입 위치
    }

    // 3) upper_bound: target 초과가 처음 나타나는 위치
    //   예) [1,2,4,4,5], target=4 -> 4
    public static int upperBound(int[] a, int target) {
        int l = 0, r = a.length; // [l, r)
        while (l < r) {
            int m = l + (r - l) / 2;
            if (a[m] <= target) l = m + 1;
            else r = m;
        }
        return l; // 삽입 위치
    }

    // 4) 존재 여부/개수 구하기: lower/upper 활용
    public static boolean exists(int[] a, int target) {
        int lb = lowerBound(a, target);
        return lb < a.length && a[lb] == target;
    }

    public static int countEqual(int[] a, int target) {
        return upperBound(a, target) - lowerBound(a, target);
    }

    // 사용 예시
    public static void main(String[] args) {
        int[] arr = {1, 2, 4, 4, 5, 7, 9};
        System.out.println(binarySearch(arr, 5));       // -> 4
        System.out.println(lowerBound(arr, 4));         // -> 2
        System.out.println(upperBound(arr, 4));         // -> 4
        System.out.println(exists(arr, 8));             // -> false
        System.out.println(countEqual(arr, 4));         // -> 2
        System.out.println("complete");
    }
}
구분주요 구성설명
Heap 영역Bean, Controller, DTO, Cache, Hibernate 엔티티 등애플리케이션 로직 대부분이 머무는 공간
Metaspace클래스 로더, 스프링 빈 정의, 프록시 클래스 등스프링의 리플렉션·프록시 기반 구조 때문에 많음
Thread StackHTTP 스레드, DB Connection Pool, Scheduler스레드 개수에 비례해 Stack 메모리 증가
Direct MemoryNetty, JDBC Driver, ByteBuffer외부 I/O 버퍼, 대용량 직렬화 처리
Code CacheJIT 컴파일된 코드서버 장기 구동 시 점진적으로 증가

실무 팁:

  • Xmx만 줄여서 “메모리 제한 걸었다”고 생각하면 안 됩니다.

실제 RSS(Resident Set Size, OS가 보는 메모리)는

Heap + Metaspace + Stack + DirectMemory의 합입니다.

[자바 프로세스 메모리와 OS 메모리의 차이]#

운영체제는 프로세스 전체의 **물리 메모리 사용량(RSS)**을 기준으로 판단하지만, JVM 내부에서는 Heap만 보고 “남았다”고 착각하는 경우가 많습니다.
